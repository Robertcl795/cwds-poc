# Design System Documentation

This repo now uses three canonical documentation tracks.

- Track A: Architecture & Vision (`docs/architecture-vision`)
- Track B: Package Reference (`docs/package-reference`)
- Track C: Release Readiness (`docs/release-readiness`)
- Governance source: `rules/*` (architecture/authoring/testing/release/contribution)

Anything outside these tracks is considered legacy implementation history and should not be used as source-of-truth.
